Introduction

The CIA World Factbook contains data about people, government, economy, communications, and more for each world entity. In this tutorial, you will learn how to curate data from The CIA World Factbook and perform analysis to answer interesting questions about countries around the world. In particular, we will be collecting data about expected years spent in school, education expenditures, literacy, and unemployment rate in each country. Then, we will use data science techniques such as exploratory analysis, visualization, linear regression, and machine learning to analyze the relationship between education and unemployment rate.

Background

Studies show that limitations in education can lead to higher unemployment rates, and that the development of education can increase employment in certain countries. Here are some background readings that investigate the relationship between education and unemployment:

  • The relationship between education and employment: A theoretical analysis and empirical test
  • Where Unemployment Is Highest, Education Is A Big Part Of Why
  • Unemployment rates and earnings by educational attainment

These studies suggest that there is a positive correlation between education and employment within a specific country such as the United States. In this project, we take our analyses to a broader scale to investigate all countries on an international level. This information could indicate whether differences in education expenditures or educational structure in a country can lead to a change in unemployment rates.

Data Curation

The CIA World Factbook contains information about each country in a separate webpage. This data is pretty messy, so it will take a bit of work to parse all of the HTML into a more readable data frame in R.

First, we will obtain a list of all the countries and their corresponding webpages. On the front page of the factbook, you will find a drop-down menu containing this information. Using the rvest package, we can parse the HTML for option nodes in this menu. This will give us a list of all the countries and their corresponding webpage.

webpage <- read_html("https://www.cia.gov/library/publications/the-world-factbook/")
countries <- webpage %>%
  html_nodes("option")

Next, we will use regular expressions and the stringr library to parse each country and its corresponding webpage. If you arent familiar with regular expressions, you can read more about them here. As an example, well start with the first alphabetical country, Afghanistan. Note that this is the third value in our countries list.

page <- str_extract(countries[3],"[a-z][a-z].html")  # use stringr::str_extract to obtain link ending "af.html"
name <- str_match(countries[3], " ([A-z,-]+) ")[,2]  # find the country name using stringr::str_match

# Read HTML from country's webpage
link <- paste("https://www.cia.gov/library/publications/the-world-factbook/geos/", page, sep="")
html <- readLines(link)

Data Parsing

Now that we have the HTML of this webpage, we need to extract specific data about education and unemployment. We will be extracting the following data for each country:

  • School life expectancy (number of years that a student is expected to stay in school)
  • Education expenditures (the % of GDP spent on education)
  • Literacy (% of people age 15+ that can read and write)
  • Unemployment Rate
  • Region

For formal definitions of these fields, see here.

# Extract school life expectancy
line <- grep('School life expectancy', html)
school_years <- ifelse(identical(line, integer(0)), NA, str_match(html[line+1], "([0-9.]+) years")[[2]])
school_years <- as.numeric(school_years)

# Extract education expenditures
line <- grep('Education expenditures', html)
education <- as.numeric(str_match(html[line+1], ">([0-9.]+)%")[,2])

# Extract literacy 
line <- grep('age 15 and over can read and write',  html)
literacy <- ifelse(identical(line, integer(0)), NA, str_match(html[line+1], "([0-9.]+)%")[[2]])
literacy <- as.numeric(literacy) / 100

# Extract unemployment rate
line <- grep('Unemployment rate', html)[1]
unemployment_rate <- str_extract(html[line + 1], "[0-9.%]+")
unemployment_rate <- as.numeric(gsub("%", "", unemployment_rate)) / 100

# Extract country's region
line <- grep("region1 geos_title", html)
region <- str_match(html[line], 'region1 geos_title [A-z]+">([A-z ]+)   <')[,2][2]

Now that we have all our data points, we can add this information as a row in a data frame.

# Create a data frame to store this information
df <- data.frame(country = c(name), school_years = c(school_years), education_exp=c(education), literacy = c(literacy), unemployment = c(unemployment_rate), region = c(region))

Handling missing values

As we can see, there are some NA values in the dataset. These are the missing values, or data that wasnt available from the CIA World Factbook. This data isnt missing at random, as some countries have more data availability than others. Additionally, the factbook contains some regions where none of these values are relevant, such as oceans or islands. The best way to handle these missing values is to simply remove them from our data set. We will remove any rows that do not contain unemployment rates, as well as rows that do not contain any of the 3 data points for education.

df <- df[!is.na(df$unemployment),]
df <- df[!is.na(df$literacy) | !is.na(df$school_years) |  !is.na(df$education_exp),]

Building a Simple Linear Regression Model

We will use the lm() function to build a linear regression model to analyze relationship between expected years in school and unemployment.

linear_model1 <- lm(unemployment ~ school_years, data=df)
linear_model1 %>% broom::tidy()
##           term    estimate   std.error statistic      p.value
## 1  (Intercept)  0.30269888 0.043375753  6.978527 7.367450e-11
## 2 school_years -0.01394175 0.003136143 -4.445510 1.626359e-05

Based on the results of our linear model, a country whose students are expected to spent 0 years in school will have an estimated unemployment rate of 30%. If the expected years in school goes up by a year, then the unemployment rate will decrease by around 1.4% on average. We can also decide whether there is a significant relationship between expected school years and unemployment based on a statistical significance test where the null hypothesis is no relationship. Since the p-value is less than 0.05, we can reject the null hypothesis. This suggests that there is a negative linear relationship between expected years spent in school and unemployment rate in a country.

Unsupervised Machine Learning

Unsupervised machine learning allows us to gain understanding of the structure of our dataset by finding hidden patterns in our data. We will be using an unsupervised method known as k-means clustering to see if we can find groups of countries that are similar to each other.

In order to use k-means clustering, we need to define the similarity between two countries in our dataset. The k-means clustering algorithm will use this to partition the dataset in k clusters of similar objects. For example, we could define similarity based on values for school life expectancy and education expenditure. We need to scale these values before passing them to the kmeans algorithm because they are measured in different units.

set.seed(1234)
df <- na.omit(df) # need to omit all missing values
clusters <- kmeans(scale(df[, 2:3]), 4, nstart = 20)

Visualizing K-means clustering

We can add the clusters as a column in the data frame and use ggplot to visualize the clusters.

df$cluster <- as.factor(clusters$cluster)
df %>%
  ggplot(aes(x=school_years, y=education_exp, color = cluster)) + 
  geom_point() + geom_text(data=df[df$cluster == 3,], aes(label=country))

We can visualize the four clusters than k-means partitions the dataset into. Clustering can also be used as a way of detecting outliers. We can see that Cluster 3 contains much fewer values than the other three clusters, meaning that these 3 values are far away from the remainder of the dataset. These three countries could potentially be considered outliers due to their high education expenditures.
